ipile
=====

This implementation is called by the pile template to save code bloat.
The other templates simply bloat.

malloc became redundant the day they invented the huge virtual memory
  space. We can now declare a huge array for each struct in the 
  program and rely on paging to avoid using physical memory until we
  write something in that part of the array. A pile is just that
  growing from the bottom up with a free list. 

The pile template takes the name of a struct and feeds corresponding 
  parameters to ipile

A further advantage is that we can refer to things by 32-bit 
  (or even 24 or 16-bit) indices rather than 64-bit pointers.
  This has a huge impact on data size, therefore page faults,
  therefore speed.

Oh and it persists in a file.

get/set/modUsr is a temporary hack which could grow into a struct
  determined by a secondary template parameter.

There's also lit/globals living in a memory-mapped file and some
  refactoring might be nice.
